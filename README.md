# 家庭经济困难学生精准认定系统

## 项目概述

家庭经济困难学生精准认定系统是一个集成化的图形界面应用程序，基于 PyQt5 开发，旨在为高校提供科学、高效的家庭经济困难学生认定解决方案。系统整合了三种核心算法（RRDA、AII、FCE-AII）和一个基于深度学习的权重自优化算法，实现了从数据录入、指标体系管理、困难度认定到权重优化的完整工作流程。

*注：在最新版本的指标体系中移除了家庭收入相关指标，RRDA算法暂不使用。*

### 系统特色

- **一体化管理平台**：集成指标体系选择、数据录入、算法运行和结果分析功能
- **多算法支持**：支持 RRDA、AII、FCE-AII 三种认定算法，可灵活选择
- **智能权重优化**：基于神经网络的自优化算法，可根据历史数据动态调整指标权重
- **可视化分析**：提供丰富的图表展示和数据分析功能
- **角色权限管理**：支持用户和管理员两种角色，管理员可使用权重优化功能

## 权重自优化算法详解

### 算法原理

权重自优化算法通过深度学习技术自动学习最优的指标权重配置。

**核心思想**：
1. 使用自定义 CustomLinear 层直接学习各指标权重
2. 融合多种评估信号（人工标注、AII 算法、FCE-AII 算法）
3. 训练完成后输出权重优化方案

**损失函数设计**：

$$
L_{total} = (1 - \alpha - \beta) \cdot L_{CE}(y, \hat{y}) + \alpha \cdot L_{CE}(y_{AII}, \hat{y}) + \beta \cdot L_{CE}(y_{FCE}, \hat{y})
$$

其中：
- $L_{CE}$：交叉熵损失函数
- $y$：真实标签（人工标注）
- $y_{AII}$：AII 算法生成的标签
- $y_{FCE}$：FCE-AII 算法生成的标签
- $\hat{y}$：模型预测结果
- $\alpha$：AII 算法优化权重（范围 [0, 1]）
- $\beta$：FCE-AII 算法优化权重（范围 [0, 1]）

**约束条件**：$\alpha + \beta \leq 1$，建议不超过 0.7，保留至少 30% 的原始标签权重以确保训练稳定性。

### 神经网络模型架构

```python
Net(
  (fc1): CustomLinear(16, 1)        # 学习 16 个指标权重
  (fc2): Linear(1, 64)              # 非线性变换层
  (bn2): BatchNorm1d(64)
  (dropout2): Dropout(0.2)
  (fc3): Linear(64, 128)
  (bn3): BatchNorm1d(128)
  (dropout3): Dropout(0.2)
  (fc4): Linear(128, 64)
  (bn4): BatchNorm1d(64)
  (dropout4): Dropout(0.2)
  (fc5): Linear(64, 4)              # 输出 4 个困难等级
)
```

**CustomLinear 层**：
- 仅包含权重参数，无偏置项
- 使用 Softmax 进行权重归一化
- 输出加权后的综合困难度分数

### 训练流程

1. **数据准备**：加载指标体系和带标签的学生数据
2. **编码处理**：将学生数据编码为数值向量
3. **生成多源标签**：
   - 真实标签：人工认定结果
   - AII 标签：基于 AII 算法计算的认定结果
   - FCE-AII 标签：基于 FCE-AII 算法计算的认定结果
4. **模型训练**：
   - 使用 Adam 优化器
   - 每个 epoch 记录损失和准确率，实时保存权重演化过程
5. **结果输出**：
   - 优化后的指标体系文件（CSV 格式）
   - 权重变化记录（weight_change.csv）
   - 可视化图表（loss.png, acc.png, weight.png）

### 权重变化显著性分析

为科学识别权重变化显著的指标，采用**联合判定法**：

**阈值设定**：
- 权重变化量阈值：$T_{\Delta w} = \overline{\Delta w} + \sigma_{\Delta w}$
- 权重变化百分比阈值：$T_{p} = \overline{p} + \sigma_{p}$

**判定标准**：同时满足以下两个条件之一，则判定为显著变化：
1. 权重变化量 $\Delta w_i > T_{\Delta w}$
2. 权重变化百分比 $p_i > T_{p}$

**方法优势**：
- 结合绝对变化和相对变化，避免小权重指标虚高
- 基于统计学原理，具有科学性和客观性
- 识别出的指标对认定结果具有实际影响

## 命令行使用

核心算法也支持命令行运行。

### 权重自优化命令行

```bash
python selfOptimization.py \
    --old_zbtx_file data/指标体系_v1.csv \
    --dataset data/raw_500_modified.csv \
    --new_zbtx_file results/optimized_index.csv \
    --lr 0.01 \
    --epochs 100 \
    --batch_size 128 \
    --alpha 0.3 \
    --beta 0.1
```

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|-----|
| `--old_zbtx_file` | str | data/指标体系_v1.csv | 原始指标体系文件路径 |
| `--new_zbtx_file` | str | None | 优化后的新指标体系文件保存路径 |
| `--dataset` | str | data/raw_500_modified.csv | 带标签的原始数据 |
| `--lr` | float | 0.01 | 学习率（建议0.001-0.1）|
| `--epochs` | int | 10 | 训练轮次 |
| `--batch_size` | int | 128 | 批次大小 |
| `--alpha` | float | 0.33 | AII算法优化权重，范围[0, 1] |
| `--beta` | float | 0.33 | FCE-AII算法优化权重，范围[0, 1] |

**注意**：
- `alpha` 与 `beta` 之和建议不超过 0.7，保留至少 30% 的原始标签权重
- 学习率过大可能导致训练不稳定，过小则收敛缓慢
- 训练数据需包含"实际认定结果"列

### 输出文件说明

**可视化输出**（保存于 `tmp/`）：
- `loss.png`：训练损失曲线
- `acc.png`：准确率变化曲线
- `weight.png`：各指标权重演化趋势（多子图，按指标分组）

**数据输出**：
- 优化后的指标体系文件（CSV 格式，包含新的权重）
- `weight_change.csv`：详细记录每个 epoch 的权重变化
- `weight_summary.csv`：权重统计汇总，包括初始权重、最终权重、变化量和变化百分比

## 常见问题 (FAQ)

### 如何修改用户登录信息？

编辑 [login_window.py](login_window.py) 文件中的 `verify_credentials` 方法，修改账号密码验证逻辑。默认账号信息保存在 `users.txt` 文件中。

### 训练时显示"数据维度不匹配"错误？

- 检查指标体系文件中的二级指标数量是否为 16 个（当前模型固定为 16 维）
- 如需支持其他维度，需修改 [model.py](model.py) 中的 `Net` 类，将 `CustomLinear(16, 1)` 改为对应维度

### 权重优化训练时间过长怎么办？

- 减少训练轮数（epochs），尝试 10-50 轮
- 增大批量大小（batch_size），如 256 或 512
- 减小学习率（lr），如 0.001，可能加快收敛
- 使用 GPU 加速（系统自动检测 CUDA，无需额外配置）

### 如何解读权重变化显著性分析结果？

- **显著增加的指标**：在优化后对困难度认定的影响更大，建议重点关注
- **显著减少的指标**：在优化后影响减弱，可能存在信息冗余或与其他指标相关性高
- **变化不显著的指标**：当前权重配置较为合理，无需大幅调整

### FCE-AII 算法与 AII 算法有何区别？

- **AII**：确定性算法，基于固定的权重和分数计算
- **FCE-AII**：考虑模糊性和不确定性，基于历史数据的统计概率
- **选择建议**：
  - 数据量少或初次使用：选择 AII
  - 有丰富历史数据且需考虑指标模糊性：选择 FCE-AII
